1. State Initialization (hooks/useDashboard.ts):
- กำหนดตัวแปร columns และ rows (Default: 20x16)
- เก็บ blocks เป็น Array ของ Object ที่มีพิกัด x, y, w, h
- จุดสังเกต: พิกัด x, y ปัจจุบันบันทึกแบบ 1-indexed (เริ่มนับ 1 ที่จุดตัดแรก)
- Storage Key: `obsidian-zen-dashboard-config`

2. Viewport & Paging Setup (Dashboard.tsx):
- สร้าง drift-viewport สำหรับการทำสไลด์หน้า (Horizontal Paging)
- ใช้ transform: translateX เพื่อเลื่อนหน้าจอตาม activePage
- โครงสร้าง: .dashboard-page > .drift-viewport > .drift-screen (multi-page)

3. Grid Visualization (DashboardGrid.tsx):
- Intersections (Dots): วนลูปสร้างจุด + ตามจำนวน (columns + 1) * (rows + 1)
- ใช้ position: absolute และคำนวณตำแหน่งด้วย % (left: (100 / columns) * c %)
- ใช้ transform: translate(-50%, -50%) เพื่อให้จุดตัดอยู่กึ่งกลางพอดี
- Coordinates (Numbers): วางตัวเลขพิกัดเขียวที่ (i + 0.5) ของ Cell เพื่อบอก index (ใช้ standalone layer)

4. Block Positioning Logic (DashboardBlock.tsx):
- คำนวณตำแหน่งด้วยสูตร %:
  left = ((x - 1) / columns) * 100%
  top = ((y - 1) / rows) * 100%
  width = (w / columns) * 100%
  height = (h / rows) * 100%
- Core Problem: การใช้ (x - 1) เป็นการพยายามแปลง 1-index ให้เป็น 0-index ในระดับ Rendering ซึ่งทำให้ตรรกะซ้ำซ้อนและ Error-prone

5. Interaction System (Drag & Resize):
- Drag: ใช้ window mousemoveListener, คำนวณ deltaPixel -> แปลงเป็น deltaCell (หาร cellWidth) -> Math.round เพื่อ Snap
- UI Feedback: เปลี่ยน cursor เป็น grab/grabbing และเพิ่ม class .dragging
- Resize: รองรับ 8 ทิศทาง (N, S, E, W, NW, NE, SW, SE) โดยส่ง resizeDir เข้าไปใน MouseDown handler

Block (Dashboard Block) - วัตถุที่จับต้องได้ดั่ง "กระถางต้นไม้"
- ขนาด สีเหลี่ยม เท่ากับ gird ที่อยู่ข้างหลัง แบบไม่มีร้อยต่อ gap 0px
- ขนาดเล็กสุดคือ 1*1 ,ใหญ่สุดไม่จำกัด 
- สามารถย้ายได้ด้วยการ drag
- สามารถปรับขนาดได้ด้วยการลากที่มุมขวาล่าง
- ไม่สามารถซ้อนกันได้ ทั้งการย้ายและการปรับขนาด
- เมื่ออยู่ใน edit mode สามารถ ปรับแต่ง+ลบ ได้ที่มุมขวาบน
- มีความเป็น Solid Object สูง คือเป็นวัตถุที่มีสถานะแข็ง ไม่สามารถวางทับซ้อนกันได้ (Collision Detection)
- การจัดวางต้องสมบูรณ์แบบ (Pixel-Perfect) เสมอ โดยการ Snap-to-Grid และกำจัด Gap ระหว่างช่องให้เป็น 0px
- ระบบความปลอดภัย: จะไม่สามารถลดจำนวนช่องตาราง (Grid) ได้จนกว่าผู้ใช้จะขยับบล็อกที่ขวางอยู่ออกไปก่อน เพื่อป้องกันบล็อกขาดหาย

Vis
- Vis คือ Visualization ที่ตั้งอยู่บน Block เพื่อ นำเสนอข้อมูลตามที่ ผู้ใช้ต้องการ
- Vis type
    - Donut Chart ใช้ดูสัดส่วนของสินทรัพย์มากกว่า 1 ประเภท
    - loading Chart ใช้ดูจำนวนของสินทรัพย์ เมื่อเทียบกับจำนวนเป้าหมาย
    - line chart ใช้ดูหรือเทียบ ข้อมูลที่มีการเปลี่ยนแปลง กับแกนเวลา
    - No chart สำหรับใช้เป็นหัวข้อ
    - picture สำหรับใช้เป็นรูปภาพ
    - checklist or note สำหรับใช้เป็นบันทึก
    - calendar สำหรับใช้เป็นปฏิทิน (ยังไม่เอา เป็นไอเดียเฉยๆ)
- vis องค์ประกอบ 3ส่วนคือ มี header ,พื้นที่สำหรับ Visualization และ config ของ vis-block ใน edit mode
    - vis-header ใช้แสดงชื่อของ vis-block
    - vis-zone พื้นที่สำหรับ Visualization ใช้แสดงข้อมูลตามที่ผู้ใช้ต้องการ
    - vis-config ของ vis-block ใน edit mode อยู่ที่มุมขวาบน ของ Block ใช้จัดการกับ vis-config สำหรับปรับแต่ง
- vis-config คือ การปรับแต่ง vis ที่จะแสดงบน block ใน vis-zone สิ่งที่ปรับได้มีดังนี้
    - vis-header ข้อความที่แสดงบน vis-header ซ้ายบน
    - vis-type ประเภทของ vis ที่จะแสดงบน vis-zone เลือกจาก vis-type ที่มีให้เลือก
    - vis-data ข้อมูลที่จะแสดงบน vis-zone โดยจะสัมพันธ์กับ vis-type เช่น ถ้าเลือกเป็น Donut chart ก็จะมีข้อมูลให้เลือกแสดงเป็นประเภทของสินทรัพย์ ถ้าเลือกเป็น loading chart ก็จะมีข้อมูลให้เลือกแสดงเป็นจำนวนของสินทรัพย์ ถ้าเลือกเป็น line chart ก็จะมีข้อมูลให้เลือกแสดงเป็นข้อมูลที่เทียบกับเป้าหมาย
