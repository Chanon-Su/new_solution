เนื่องจาก window context ที่ใช้เป็นเหมือนความจำระยะสั้นของ agents นั้น เมือมีการสนทนาไปเรื่อยๆ จะทำให้เกิดการเก็บข้อมูลใหม่แล้วข้อมูลเก่าก็จะถูกดันออกไปหายไป ทำให้คุณภาพงานหรือการโฟกัส การคิดคำนึงถึงเนื้องาน หายไปตาม window context ที่หายไปเหล่านั้น, workflows นี้จึงออกแบบมาเพื่อ reset window context ตามที่เคยตั้งไว้ + ให้สามารถ resume งานได้ทำเอาไว้ก่อนหน้านี้ได้ ทำให้การทำงานเป็นไปอย่างต่อเนื่อง



1. สรุปความคืบหน้าของ Agent แต่ละคน เอาไว้ชั่วคราวก่อน เพื่อรอการ reset window context (ถ้ามี) (ถ้าเป็น new conversation ก็ไปทำขั้นที่ 2 เลย)
2. ให้ Agent ทุกคนอ่านทบทวน .core ทั้งหมด เพื่อทบทวน window context เพื่อรักษาระดับคุณภาพของการทำงาน (reset window context) // turbo
3. กลับมาทำงานต่อ โดยอ่าน+ทำงาน ตามที่ สรุปความคืนหน้าเอาไว้ ก่อนหน้านี้ (ถ้ามี) (ถ้าเป็น new conversation ก็ให้เตรียมพร้อมรอรับงานใหม่ที่จะมา)


**กฎเหล็กเรื่องภาษา:** ในการสื่อสารทุกขั้นตอน ไม่ว่าจะเป็นการตอบโต้ในแชท หรือการสร้าง/อัปเดตเอกสารงานทั้งหมด (เช่น implementation_plan, task, walkthrough, report) **ต้องใช้ภาษาไทยเป็นหลัก 100%** เพื่อความถูกต้อง แม่นยำ และรักษาคุณภาพการสื่อสารตามความตั้งใจของผู้ใช้ (ยกเว้นส่วนที่เป็น Technical Term หรือ Code)
