0. workflowนี้ @V เป็นคนจัดการหลัก และให้ agentที่เหลือ เป็นคนแนะนำเท่านั้น ว่าควรจะจัดเก็บหรือระเบียบเป็นแบบไหน แต่ละ file แต่ละ role ก็มีบทบาทการจัดเก็บที่แตกต่างกัน แต่ V จะเป็นคนตัดสินใจเอง
1. **สำรวจไฟล์ทั้งหมด (Inventory):**
   - รันคำสั่ง list ทุกไฟล์ใน `.core_universal` และ `.core_current_project`
   - ค้นหาไฟล์ .md ใน project root ที่ขึ้นต้นด้วย `.core` หรืออยู่ใน `docs/`, `.wisdom/`

2. **จำแนกไฟล์ (Classification):**
   - แยกไฟล์เข้า 6 หมวดหมู่ (Category A-F) ตาม `PROMPT_CORE_REBUILD.md`

3. **ดำเนินการย้าย (Migrate):**
   // turbo
   - สร้าง Folder ที่ยังไม่มี
   // turbo
   - ย้ายไฟล์ตามหมวดหมู่ที่จำแนกไว้
   - ห้ามลบไฟล์ต้นฉบับจนกว่าจะตรวจสอบความถูกต้องเสร็จสิ้น (จะลบไฟล์หรือไม่ลบไฟล์ ให้ยืนยันกับผู้ใช้งานก่อน)

4. หลังจาก จัดระเบียบ .core ทั้งสองเสร็จ ให้ดูใน โฟลเดอร์อื่นของทั้งโปรเจคเลย เช่น src
   - มีไฟล์ไหนที่ควรจะย้ายไฟล์ตามหมวดหมู่ที่จำแนกไว้
   - ถ้าไฟล์ไหนที่อยู่ในหมดวหมู่แล้ว แต่ยังดูไม่เรียบร้อยหรือไม่เป็นแบบแผน ก็อาจจะ สร้างโฟลเดอร์มาเก็บให้ดูเป็นระเบียบมากขึ้น(// turbo) แล้วย้ายไฟล์เหล่านั้นให้อยู่ในโฟลเดอร์ใหม่
   - ห้ามลบไฟล์ต้นฉบับจนกว่าจะตรวจสอบความถูกต้องเสร็จสิ้น (จะลบไฟล์หรือไม่ลบไฟล์ ให้ยืนยันกับผู้ใช้งานก่อน)

5. สุดท้ายคือทำ map.md เอาไว้ใน .core เพื่อลดภาระ ที่จะต้องให้ agent ไปไล่หาว่ามีไฟล์อะไรอยู่ที่ไหนบ้าง โดยที่ยังไม่ต้องอ่านทุกโฟลเดอร์ หลายๆครั้งที่ agent ต้องไปไล่ค้นทุกโฟลเดอร์เพื่อหาว่า ข้อมูลอยู่ไหน แต่ถ้าใช้ map.md จะรู้ได้เลยว่าอะไรอยู่ที่ไหน แล้วมุ่งไปหาที่ตรงนั้นพอ ประหยัด quota, ประหยัด เวลา, ประหยัด พลังงานด้วย

ปล. ถ้าไม่แน่ใจหรือมีทางเลือกในการเก็บหรือจำแนก ให้ถามผู้ใช้ได้เลย ไม่มีถูกหรือผิด มีแต่ ผู้ใช้ชอบหรือไม่ และ มีประสิทธิภาพมั้ย?
